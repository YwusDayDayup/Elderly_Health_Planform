# 慧养家园 - 系统架构文档

## 1. 系统概述

### 1.1 项目简介
慧养家园是一个基于 Spring Boot 3 + Vue 3 构建的智慧养老健康服务管理平台，支持老人健康档案管理、家庭绑定、服务预约、位置安全监控、社区互动等功能。

### 1.2 设计目标
- **高可用性**：采用分布式会话管理，支持水平扩展
- **安全性**：基于 Sa-Token 的认证授权体系，支持多角色权限控制
- **可维护性**：前后端分离架构，模块化设计，代码规范统一
- **易扩展性**：采用 MyBatis-Plus、MapStruct 等工具简化开发，支持快速迭代

#### 3.1.2 权限控制策略

**角色定义**
- `ADMIN`：管理员 - 全局管理权限
- `ELDER`：老人 - 查看个人信息、预约服务
- `FAMILY`：家属 - 查看绑定老人信息、代为预约
- `STAFF`：服务人员 - 接单、处理订单

**路由权限配置**
```java
// SaTokenConfig.java
SaRouter.match("/api/admin/**", r -> {
    StpUtil.checkLogin();           // 必须登录
    checkAdminRole();               // 必须是 ADMIN 角色
})
.match("/api/statistics/**", r -> {
    StpUtil.checkLogin();
    checkAdminRole();
})
.match("/**")
.notMatch(whitelist)                // 排除白名单
.check(StpUtil::checkLogin);        // 其他路径需登录
```

**白名单路径**（无需认证）
- `/api/public/**` - 公开接口（登录、注册、公开内容）
- `/swagger-ui/**` - API 文档
- `/uploads/**` - 静态文件访问

#### 3.1.3 Token 配置
```yaml
sa-token:
  token-name: Authorization        # Token 名称
  token-prefix: Bearer             # Token 前缀
  timeout: 86400                   # 24小时过期
  token-style: uuid                # UUID 风格
  is-concurrent: true              # 允许并发登录
```

```java

    private boolean success;      // 是否成功

    private String message;       // 提示信息

}
```

**成功响应示例**
```json

  "code": "0",
```

**失败响应示例**

#### 3.3.2 异常处理机制

**全局异常处理器 (GlobalExceptionHandler)**

| 异常类型 | HTTP 状态码 | 业务码 | 说明 |
|---------|------------|--------|------|
| `NotLoginException` | 401 | 401 | 未登录或 Token 过期 |
| `NotRoleException` | 403 | 403 | 无权限访问 |
| `BizException` | 400/404/500 | 自定义 | 业务异常 |
| `MethodArgumentNotValidException` | 400 | 400 | 参数校验失败 |
| `NoResourceFoundException` | 404 | 404 | 接口不存在 |
| `Exception` | 500 | 500 | 未知异常 |

**异常处理流程**
```
请求 → Controller → Service (抛出异常)
                        ↓
            GlobalExceptionHandler 捕获
                        ↓
            转换为统一 Result 响应
                        ↓
            返回给前端 (带 HTTP 状态码)
```

```typescript
router.beforeEach((to, from, next) => {
  NProgress.start()
  const token = localStorage.getItem('token')
  
  // 公开路径直接放行
  if (to.meta.public) {
    next()
  } 
  // 未登录跳转到登录页
  else if (!token) {
    next('/login')
  } 
  // 已登录放行
  else {
    next()
  }
})
```

### 4.4 HTTP 请求封装

**请求拦截器**
```typescript
// 请求拦截：注入 Token
request.interceptors.request.use(config => {
  const token = localStorage.getItem('token')
  if (token) {
    config.headers.Authorization = `Bearer ${token}`
  }
  return config
})

// 响应拦截：统一错误处理
request.interceptors.response.use(
  response => response.data,
  error => {
    if (error.response?.status === 401) {
      // Token 过期，跳转登录
      localStorage.removeItem('token')
      router.push('/login')
    }
    ElMessage.error(error.response?.data?.message || '请求失败')
    return Promise.reject(error)
  }
)
```

## 7. 安全设计

### 7.1 认证安全
- **密码加密**：使用 BCrypt 算法（Spring Security Crypto）
- **Token 机制**：UUID 风格，存储在 Redis，支持过期时间
- **HTTPS**：生产环境强制使用 HTTPS

### 7.2 权限控制
- **路由级别**：Sa-Token 拦截器统一验证
- **角色检查**：管理员接口额外检查 ADMIN 角色
- **数据隔离**：用户只能访问自己的数据（通过 userId 过滤）

### 7.3 输入验证
- **参数校验**：使用 Spring Validation 注解
- **SQL 注入防护**：MyBatis-Plus 预编译语句
- **XSS 防护**：前端 Element Plus 自动转义

### 7.4 日志审计
- **请求日志**：`RequestLogFilter` 记录所有请求
- **操作日志**：`t_op_log` 记录关键操作
- **异常日志**：`GlobalExceptionHandler` 记录所有异常

- **异常处理**：统一使用 `BizException`，不直接抛出 `RuntimeException`

### 10.2 Git 规范
```
feat: 新功能
fix: 修复 Bug
docs: 文档更新
style: 代码格式调整
refactor: 重构
test: 测试相关
chore: 构建/工具链相关
```

### 10.3 接口规范
- **RESTful 风格**：GET（查询）、POST（创建）、PUT（更新）、DELETE（删除）
- **统一前缀**：`/api`
- **统一响应**：`Result<T>` 结构
- **错误码**：使用 HTTP 状态码 + 业务码

## 12. 常见问题

### 12.1 Token 过期处理
**问题**：用户操作时 Token 过期
**解决**：前端拦截 401 响应，自动跳转登录页

### 12.2 跨域问题
**问题**：开发环境前后端分离导致跨域
**解决**：Vite 配置代理 `proxy: { '/api': 'http://localhost:8080' }`

### 12.3 数据库连接失败
**问题**：启动时报 `Unknown database`
**解决**：手动创建数据库 `CREATE DATABASE 154laoren`

### 12.4 Redis 连接失败
**问题**：启动时报 Redis 连接异常
**解决**：确保 Redis 服务已启动
