# 快速部署指南

本文档适用于收到项目交付包（后端 JAR + 前端源码）的用户，帮助快速部署运行系统。

---

## 📦 交付包内容

```
项目交付包/
├── backend/
│   └── huiyang-home-1.0.0.jar          # 后端可执行 JAR 包
├── frontend/                            # 前端源码目录
│   ├── src/                            # 源代码
│   ├── public/                         # 静态资源
│   ├── package.json                    # 依赖配置
│   ├── vite.config.ts                  # 构建配置
│   └── ...
└── 快速部署指南.md                      # 本文档
```

## 🚀 部署步骤

### 第一步：准备数据库

1. **启动 MySQL 服务**

2. **创建数据库**
```sql
CREATE DATABASE IF NOT EXISTS `154laoren` 
  DEFAULT CHARACTER SET utf8mb4 
  COLLATE utf8mb4_unicode_ci;
```

> 💡 **重要说明**：
> - 数据库表结构会在后端首次启动时**自动创建**（使用 Flyway 自动迁移）
> - 系统会自动初始化**演示数据**，包括测试账号和示例业务数据
> - **无需手动导入 SQL 文件**，只需创建空数据库即可

3. **修改数据库配置（如果需要）**

如果你的 MySQL 用户名/密码不是默认的 `root/123456`，需要修改配置：

创建 `application-prod.yml` 文件（与 JAR 包同目录）：

```yaml
spring:
  datasource:
    url: jdbc:mysql://localhost:3306/154laoren?useUnicode=true&characterEncoding=utf-8&serverTimezone=GMT%2B8
    username: 你的用户名
    password: 你的密码
  data:
    redis:
      host: localhost
      port: 6379
      password: Redis密码（如果有）
```

### 第二步：启动 Redis

```bash
# Windows
redis-server

# Linux/Mac
sudo systemctl start redis
# 或
redis-server
```

### 第三步：启动后端

打开命令行，进入 `backend` 目录：

**Windows (PowerShell):**
```powershell
# 如果有自定义配置文件
java -jar huiyang-home-1.0.0.jar --spring.config.location=application-prod.yml

# 如果使用默认配置
java -jar huiyang-home-1.0.0.jar
```

**Linux/Mac:**
```bash
# 如果有自定义配置文件
java -jar huiyang-home-1.0.0.jar --spring.config.location=application-prod.yml

# 如果使用默认配置
java -jar huiyang-home-1.0.0.jar
```

**看到以下信息表示启动成功：**
```
应用已启动  local=http://localhost:8080/  external=http://x.x.x.x:8080/
```

启动过程中会看到 Flyway 执行数据库迁移的日志：
```
Flyway: Migrating schema `154laoren` to version "1 - init"
Flyway: Migrating schema `154laoren` to version "2 - care domain"
Flyway: Migrating schema `154laoren` to version "3 - community post"
```

这表示数据库表结构和演示数据已自动创建完成。

> 💡 **提示**：保持此命令行窗口打开，关闭会停止后端服务

### 第四步：部署前端

**打开新的命令行窗口**，进入 `frontend` 目录：

#### 4.1 安装依赖（首次部署需要）

```bash
npm install
```

> ⏱️ 首次安装可能需要 3-5 分钟，请耐心等待

#### 4.2 启动前端开发服务器

```bash
npm run dev
```

**看到以下信息表示启动成功：**
```
VITE v7.3.2  ready in xxx ms
➜  Local:   http://localhost:5173/
```

> 💡 **提示**：保持此命令行窗口打开，关闭会停止前端服务

## 📊 数据库说明

### 自动初始化机制

系统使用 **Flyway** 进行数据库版本管理，后端首次启动时会自动执行以下操作：

1. **创建表结构**
   - 用户表（t_user、t_user_role）
   - 老人档案表（t_elder_profile）
   - 服务人员表（t_staff_profile）
   - 家属绑定表（t_family_binding）
   - 服务相关表（t_service_category、t_service_project、t_service_order）
   - 健康管理表（t_health_record、t_health_alert）
   - 位置安全表（t_safe_zone、t_location_record）
   - 社区互动表（t_community_post）
   - 内容管理表（t_content）
   - 消息表（t_msg）
   - 文件表（t_file）
   - 日志表（t_op_log、t_external_call_log）

2. **初始化演示数据**
   - 管理员账号：admin / 123456
   - 测试老人账号：demo_elder_1 ~ demo_elder_5 / 123456
   - 测试服务人员：demo_staff_1 ~ demo_staff_3 / 123456
   - 测试家属账号：demo_family_1 ~ demo_family_3 / 123456
   - 示例服务分类和项目
   - 示例健康记录
   - 示例服务订单
   - 示例社区帖子

### 迁移脚本位置

数据库迁移脚本位于 JAR 包内部：
```
BOOT-INF/classes/db/migration/
├── V1__init.sql              # 基础表结构
├── V2__care_domain.sql       # 养老业务表 + 演示数据
└── V3__community_post.sql    # 社区帖子表
```

### 重置数据库

如果需要重置数据库到初始状态：

```sql
-- 删除数据库
DROP DATABASE IF EXISTS `154laoren`;

-- 重新创建
CREATE DATABASE IF NOT EXISTS `154laoren` 
  DEFAULT CHARACTER SET utf8mb4 
  COLLATE utf8mb4_unicode_ci;

-- 重启后端服务，会自动重新初始化
```

### 禁用演示数据

如果不需要演示数据，可以在配置文件中禁用：

```yaml
app:
  seed:
    enabled: false  # 设置为 false 禁用演示数据生成
```

## 📝 生产环境部署（可选）

如果需要部署到生产服务器，而不是本地开发：

### 方式一：使用 Nginx 部署前端（推荐）

#### 1. 构建前端生产包

```bash
cd frontend
npm run build
```

生成的文件在 `frontend/dist/` 目录

#### 2. 配置 Nginx

创建 Nginx 配置文件：

```nginx
server {
    listen 80;
    server_name your-domain.com;

    # 前端静态文件
    root /path/to/frontend/dist;
    index index.html;

    # 前端路由
    location / {
        try_files $uri $uri/ /index.html;
    }

    # 后端 API 代理
    location /api/ {
        proxy_pass http://localhost:8080;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }

    # Swagger 文档
    location /swagger-ui/ {
        proxy_pass http://localhost:8080;
    }
}
```

#### 3. 后台运行后端

**Linux (使用 nohup):**
```bash
nohup java -jar huiyang-home-1.0.0.jar > app.log 2>&1 &
```

**Windows (使用 start):**
```cmd
start /B java -jar huiyang-home-1.0.0.jar
```

### 方式二：前端集成到 JAR（单体部署）

如果想要一个 JAR 包同时包含前后端：

#### 1. 构建前端
```bash
cd frontend
npm run build
```

#### 2. 将前端文件复制到后端
```bash
# Windows PowerShell
Remove-Item -Recurse -Force backend/src/main/resources/static -ErrorAction SilentlyContinue
New-Item -ItemType Directory -Path backend/src/main/resources/static -Force
Copy-Item -Recurse -Force frontend/dist/* backend/src/main/resources/static/

# Linux/Mac
rm -rf backend/src/main/resources/static
mkdir -p backend/src/main/resources/static
cp -r frontend/dist/* backend/src/main/resources/static/
```

#### 3. 重新打包后端
```bash
cd backend
mvn clean package -DskipTests
```

#### 4. 运行单体 JAR
```bash
java -jar huiyang-home-1.0.0.jar
```

访问地址变为：http://localhost:8080/

## ❓ 常见问题

### Q1: 后端启动报错 "Unknown database '154laoren'"

**解决方案：** 需要先创建数据库
```sql
CREATE DATABASE IF NOT EXISTS `154laoren` 
  DEFAULT CHARACTER SET utf8mb4 
  COLLATE utf8mb4_unicode_ci;
```

### Q2: 后端启动报错 "Redis connection failed"

**解决方案：** 
1. 检查 Redis 是否启动：`redis-cli ping`（应返回 PONG）
2. 如果未启动，执行 `redis-server` 或 `sudo systemctl start redis`

### Q3: 前端 npm install 失败

**解决方案：**
```bash
# 清除缓存重试
npm cache clean --force
npm install

# 或使用国内镜像
npm install --registry=https://registry.npmmirror.com
```

### Q4: 前端启动后无法访问后端 API

**检查项：**
1. 后端是否正常启动（访问 http://localhost:8080/swagger-ui/index.html）
2. 检查前端配置文件 `frontend/vite.config.ts` 中的代理配置

### Q5: 端口被占用

**后端端口 8080 被占用：**
```bash
# 修改启动命令
java -jar huiyang-home-1.0.0.jar --server.port=8081
```

**前端端口 5173 被占用：**
```bash
# 修改 vite.config.ts 中的 server.port 配置
# 或使用命令行参数
npm run dev -- --port 5174
```

### Q6: 找不到 JDK 21

项目要求 **JDK 17 或更高版本**（推荐使用 JDK 21）。

**下载地址：**
- Oracle JDK 21: https://www.oracle.com/java/technologies/downloads/#java21
- OpenJDK 21: https://adoptium.net/
- Oracle JDK 17: https://www.oracle.com/java/technologies/downloads/#java17

**安装后设置环境变量：**

Windows:
```powershell
$env:JAVA_HOME = "C:\Program Files\Java\jdk-21"
$env:PATH = "$env:JAVA_HOME\bin;$env:PATH"
```

Linux/Mac:
```bash
export JAVA_HOME=/path/to/jdk-21
export PATH=$JAVA_HOME/bin:$PATH
```

**验证安装：**
```bash
java -version
# 应显示 java version "17.x.x" 或 "21.x.x"
```

### Q7: 如何停止服务

**停止后端：**
- 在运行 JAR 的命令行窗口按 `Ctrl + C`

**停止前端：**
- 在运行 npm dev 的命令行窗口按 `Ctrl + C`

**后台运行的服务：**
```bash
# Linux - 查找进程
ps aux | grep huiyang-home
kill <进程ID>

# Windows - 查找进程
tasklist | findstr java
taskkill /PID <进程ID> /F
```
